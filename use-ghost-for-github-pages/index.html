Use Ghost for GitHub Pages

There are a lot of great options for creating static sites to use with GitHub Pages. The "go to" seems to be Jekyll. But, some of use like other options much better. Like Ghost! The only problem is Ghost isn't a static site generator like Jekyll. Typically you need to install Ghost on your server to use it. But, there is a way around that!

This site is actually a Ghost site. So, that is what I'm going to go over today.

Requirements

  • Linux - you can adjust this guide to fit what distro you like best, but I'm going to be making it using Ubuntu.
  • An active internet connection.
  • The ability to understand me. This is by far the most difficult requirement to meet. Because, I'm not very good with words.

Setup and Install the Prerequisits

There are a few things we will need to set up first. The first thing we are going to cover is getting our workspace setup.

Install Node.js

Ghost is written in Node.js so we will need to install the Node.js runtime. As of right now Ghost doesn't support Node.js v13, so we will need to install v12. Luckily it's a super simple process.

Just enter the following commands into your terminal:

curl -sL https://deb.nodesource.com/setup_12.x | sudo -E bash -

sudo apt-get install -y nodejs

Node.js and npm should now be installed, and ready to go!

Install Ghost Locally

Now that we have Node.js and npm we can install Ghost locally. The first thing we want to do is install the Ghost CLI tool.

npm install ghost-cli -g

Then we will make a Ghost directory, cd into it, and install Ghost by running the following commands:

mkdir ghost

cd ghost

ghost install local

This should install ghost, and start it for you automatically. If it doesn't you can run this command:

ghost start

Use your browser to navigate to http://localhost:2368/ghost to create your account, and access the admin panel. From here you can start reading the get started posts, adjust your settings, and start writting your own blog posts.

Prepare Your Static Folder

For this I'm going to assume you've already created your username.github.io repo. I created my static folder inside of my ghost folder. It just felt a lot easier for me. What we are going to do is clone our username.github.io repo to a folder named static. By running a command like this:

git clone git@github.com:username/username.github.io.git static

Generate Your Static Site Using Buster and Deploy it to GitHub Pages

Now you have Node.js, npm, and Ghost installed. You have a static folder, and a blog post or two ready. The next step is to generate your static site.

It's as simple as

buster generate --domain=http://localhost:2368

This will generate a static site into your static folder. There are a couple ways to deploy your site. The first one is:

buster deploy

However, I had a few issues with this. So, I decided to add, commit, and push the files in my static folder to GitHub myself.

Writing New Blog Posts

If you want to change something, or publish new posts, it's pretty simple. Make sure Ghost is running, and if it isn't run ghost start, make your changes, generate your static site, and then deploy it like before.

Make just you cd into your ghost folder before you generate your static site.

That's it. Now you have a working Ghost static site to use for GitHub Pages!

Bonus: GhostBuster

I got tired of typing the buster generate command, since deploy was giving me issues. And, I may have been a little bored. So I made a little script to make it easier. Here is the code:

#!/bin/bash
#
# Just a super simple bash script that uses buster
# on my local Ghost install to generate a static
# site for GitLab or GitHub
#
# Of Course it has to look like a badass hacker made it
# So it has to have ASCII at the start
#

echo " "
echo "   _____ _               _   ____            _              "
echo "  / ____| |             | | |  _ \          | |             "
echo " | |  __| |__   ___  ___| |_| |_) |_   _ ___| |_ ___ _ __   "
echo " | | |_ | '_ \ / _ \/ __| __|  _ <| | | / __| __/ _ \ '__|  "
echo " | |__| | | | | (_) \__ \ |_| |_) | |_| \__ \ ||  __/ |     "
echo "  \_____|_| |_|\___/|___/\__|____/ \__,_|___/\__\___|_|     "
echo " "
echo " "
echo "GhostBuster is getting ready to run. It will generate a static version"
echo "of your local Ghost install. Then you can deploy it to GitHub Pages."

# Pause it for 5 seconds so it can be read.
sleep 7s

# Run the buster command to generate and deploy static
# site to GitHub Pages.
buster generate --domain=http://localhost:2368

# Everything needs a completion message!!
echo "GhostBuster has finished generating your static site!"

Honestly, the important part is at the very bottom... but, still don't you want it to look all hacker and have ASCII? And, I used echo instead of printf because I wanted that GhostBuster ASCII to look right. With this script I can just type ghostbuster and generate the static site.

This isn't really needed or important. I just really enjoy running it.
